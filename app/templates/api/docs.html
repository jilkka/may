Authentication

All API requests require authentication via an API key

Generate an API key from Settings → Integrations. Include your API key in every request using one of these methods:

Option 1: Authorization Header (Recommended)

Authorization: Bearer may_your_api_key_here

Option 2: X-API-Key Header

X-API-Key: may_your_api_key_here

Example Request

curl -X GET "{{ request.host_url }}api/v1/vehicles" \
  -H "Authorization: Bearer may_your_api_key_here"

Base URL

{{ request.host_url }}api/v1

POST /vehicles

Create a new vehicle

Request Body

Field Type Required Description
namestringYesDisplay name for the vehicle
vehicle_typestringYescar, van, motorbike, or scooter
makestringNoManufacturer (e.g., Toyota)
modelstringNoModel name (e.g., Corolla)
yearintegerNoModel year
registrationstringNoLicense plate number
vinstringNoVehicle Identification Number
fuel_typestringNopetrol, diesel, electric, hybrid, lpg, cng
tank_capacitynumberNoFuel tank capacity in liters

Example

curl -X POST "{{ request.host_url }}api/v1/vehicles" \
  -H "Authorization: Bearer may_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Family Car",
    "vehicle_type": "car",
    "make": "Honda",
    "model": "Civic",
    "year": 2022,
    "fuel_type": "petrol"
  }'
PATCH /vehicles/{id}

Update a vehicle. Only include fields you want to change.

DELETE /vehicles/{id}

Delete a vehicle and all associated fuel logs and expenses. This action cannot be undone.

POST /vehicles/{id}/fuel

Create a fuel log

Request Body

Field Type Required Description
datestringYesDate in YYYY-MM-DD format
odometernumberYesCurrent odometer reading
volumenumberNoFuel volume in liters
price_per_unitnumberNoPrice per liter
total_costnumberNoTotal cost (auto-calculated if not provided)
is_full_tankbooleanNoWas tank filled completely? (default: true)
is_missedbooleanNoMark as missed fill-up (default: false)
stationstringNoGas station name
notesstringNoAdditional notes

Example

curl -X POST "{{ request.host_url }}api/v1/vehicles/1/fuel" \
  -H "Authorization: Bearer may_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "date": "2024-01-20",
    "odometer": 45000,
    "volume": 45.5,
    "price_per_unit": 1.45,
    "is_full_tank": true,
    "station": "Shell"
  }'

Error Handling

All errors return a consistent JSON format

Error Response Format

{
  "error": "Human-readable error message",
  "code": "error_code"
}

HTTP Status Codes

Code Error Code Description
200-Success
201-Created successfully
400invalid_request, validation_errorBad request or validation failed
401missing_api_key, invalid_api_keyAuthentication required or failed
403forbiddenPermission denied
404not_foundResource not found

Rate Limiting

Currently, there are no rate limits on the API. Please be considerate with your request frequency.
